Tutorial objective

Coinbase does not have a command line interface, most of Coinbase Exchange transactions are through their website. For this tutorial I will cover the Coinbase Exchange API and expand on Authentication, Accounts, Orders, Transfers, and Market data. I will mainly go through each topic I have listed and describe the command as well as the returned JSON. While the official Coinbase Exchange documentation listed here is very good, my goal is to provide only content that will allow you to interact with the Coinbase Exchange API at a basic level. Please reference the official documentation if you ever need supplemental information.



Authentication


CB Access

The Coinbase Exchange API uses 4 unique access strings in the header of each REST request.

  • CB-ACCESS-KEY: Your unique API key.
  • CB-ACCESS-SIGN: Base64 encoded sha256 HMAC signature.
  • CB-ACCESS-TIMESTAMP: Timestamp for your request in Unix Epoch time.
  • CB-ACCESS-PASSPHRASE: A unique passphrase that is set during API key creation.

All of the REST requests to Coinbase must be in "Application/JSON" format as show to the right. Get your required access strings from Account settings .

An example of an API call using Curl on Linux would need look like this:


curl --header ‘CB-ACCESS-KEY: <ACCESS KEY>, \
  CB-ACCESS-SIGN: <ACCESS SIGN>, \
  CB-ACCESS-TIMESTAMP: <ACCESS TIMESTAMP>, \
  CB-ACCESS-PASSPHRASE: <ACCESS PASSPHRASE>’ \
  https://api.exchange.coinbase.com/

While this particular command will not work it is the basis for everyone of the following examples and must be included and in further examples will just list it as CB-AUTH.


Accounts


List Accounts

You can attach an unlimited amount of accounts to your Coinbase Exchange but they are trading accounts only, which are different from just the regular Coinbase account. It is often useful to check the balance of each trading account, and this can be done with the API.

curl --header ‘CB-AUTH’ \
  https://api.exchange.coinbase.com/accounts


The command listed above will list the status of all of the trading accounts that are registered to you, however, if you are trying to see only one account you may add the account ID to the URL as shown:

curl --header ‘CB-AUTH’ \
  https://api.exchange.coinbase.com/accounts/<ACCOUNT-ID>

This returns an array of every account that is registered to your Coinbase Exchange account. Within the returned JSON each account will have an id that can be used latter to uniquely identify the account. Each account's balance will be listed and the available will be the funds able to trade.

[

        "balance": "4.520",

        "available": "100.00",
        "currency": "USD"
    }
    {
        "id": "a1b2c3df",
        "balance": "1.100",
        "hold": "0.100",
        "available": "1.00",
        "currency": "USD"
    }
]

Account History

Each Coinbase Exchange account keeps track of it recent trades and is accessible through the API with account history. Every trade an account makes either increases or decreases the accounts balance. You can check the history of an account using the command below.

curl --header ‘CB-AUTH’ \
  https://api.exchange.coinbase.com/accounts/<ACCOUNT-ID>/ledger


If you would like to retrieve account history between a certain time range you can use Pagination, for example you want to get the last 30 trades before trade id 67 you could do this.

curl --header ‘CB-AUTH’ \
  https://api.exchange.coinbase.com/accounts/<ACCOUNT-ID>/\
  ledger?before=67&limit=30

The ledger added to the REST API URL will return the latest trade made by the account identified by ACCOUNT-ID. The resulting JSON that is returned will be as formatted below. 

[
    {
        "id": "100",
        "created_at": "2014-11-07T08:19:27.028459Z",
        "amount": "0.001",
        "balance": "239.669",
        "type": "fee",
        "details": {
            "order_id": "d50ec984-77a8-460a-b958-66f114b0de9b",
            "trade_id": "74",
            "product_id": "BTC-USD"
        }
    }
]

Orders


Creating a New Order

Orders come in 3 different types Limit, Market, and Stop. Orders can only be created if the supplying account has adequate funds. Once a trade is made the requested funds will be put on hold until the transaction is finalized. To buy 4 bitcoin from the exchange at $420 USD you will send a command as follows.

curl --header ‘CB-AUTH’ \
 --data-binary '{"type":"limit", "side":"buy", \
  "product_id":"BTC-USD","price":"420", "size":"4"}' \
 --request POST \
 https://api.exchange.coinbase.com/orders


The result of a successful trade will only return an order id.

{
    "id": "0428b97b-bec1-429e-a94c-59992926778d"
}



List Orders

Every trade an account makes is strode in the public ledger. To list the open orders, either open or un-settled orders, the following command is used. curl --header ‘CB-AUTH’ \
 https://api.exchange.coinbase.com/orders

As you can see from the example this transaction has not been settled yet. By default if no status query is defined it will return open orders but you can retrieve all orders that are done with the following. curl --header ‘CB-AUTH’ \
 https://api.exchange.coinbase.com/orders?status=done

The open orders are returned in a JSON object that includes the following. 

[
    {
        "id": "d50ec984-77a8-460a-b958-66f114b0de9b",
        "size": "3.0",
        "price": "100.23",

        "status": "open",

        "fill_fees": "0.001",
        "settled": false,
        "side": "buy",
        "created_at": "2014-11-14T06:39:55.189376Z"
    }
]

Transfer Funds


Deposit/Withdraw

Once your accounts begin to trade more it is useful to be able to transfer funds between Coinbase Exchange trading accounts. This is accomplished with a POST command as shown.

curl --header ‘CB-AUTH’ \
 --data-binary '{"type":"deposit", "amount":"10.32", \
  "coinbase_account_id":"b5344d81b42e65f81af9ceba"}' \
 --request POST \
 https://api.exchange.coinbase.com/transfers

A very similar command is used to withdraw funds from an account.

curl --header ‘CB-AUTH’ \
 --data-binary '{"type":"withdraw", "amount":"1.25", \
  "coinbase_account_id":"b5145151b42e65f81af9ceba"}' \
 --request POST \
 https://api.exchange.coinbase.com/transfers


You are able to withdraw funds to any of your active Coinbase accounts outside of the exchange, including vaults. However you do have a daily limit on the amount of funds you are able to withdraw in one day.

Market Data


Trades

As I showed in the Orders section to show your own trades, you can also access every trade that is being transacted on the Coinbase Exchange network.

curl --header ‘CB-AUTH’ \
 https://api.exchange.coinbase.com/products/\
 BTC-USD/trades?limit=2

The out come is a JSON list with the most recent trades with the limits provided via pagination.

24HR Stats

It is often times very useful to analyze the market trend. With the use of 24 hour stats you are able to retrieve the last 24 hours high, low, open, and volume.

curl --header ‘CB-AUTH’ \
 https://api.exchange.coinbase.com/products/\
 BTC-USD/stats

The stats command will tell you the opening price, the high price, the low price, and the volume of trades in JSON format. 

                                        
{
    "open": "34.19000000",
    "high": "95.70000000",
    "low": "7.06000000",
    "volume": "2.41000000"
}
